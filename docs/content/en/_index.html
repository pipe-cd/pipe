PipeCD provides a continuous delivery solution that enables deployment operations (rollout, scale, rollback...) by Pull Request for declarative Kubernetes, Serverless, and Infrastructure applications.

{{< /blocks/section >}}
{{< blocks/value title="Visibility" image="visibility" image_position="left" >}}
  • Deployment pipeline UI shows clarify what is happening
  • Separate detailed logs view for each individual deployment
  • Realtime visualization of application state
  • Deployment notifications/alerts to slack, webhook
  • Insight shows the delivery performance
{{< /blocks/value >}} {{< blocks/value title="Automation" image="automation" image_position="right" >}}
  • Automated Rollback
  • Automated Deployment Analysis (based on metrics, logs, http requests...)
  • Automated Configuration Drift Detection
{{< /blocks/value >}} {{< blocks/value title="Secure" image="secure" image_position="left" >}}
  • Support single sign-on (SSO) and role-based access control (RBAC)
  • Credentials are not exposed outside cluster
  • Piped is doing only outbound requests and able to run inside a restricted/private network
{{< /blocks/value >}} {{< blocks/value title="Multi-provider & Multi-Tenancy" image="multi-provider" image_position="right" >}}
  • Support multi-cloud: Kubernetes, Terraform, CloudRun, Lambda...
  • Easy to operate multi-cluster, multi-tenancy.
  • Support multi-strategy: rolling update, canary, bluegreen.
