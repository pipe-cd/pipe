WIP PipeCD provides a unified continuous delivery solution that enables deployment operations (rollout, scale, rollback...) by Pull Request for declarative Kubernetes, Serverless, and Infrastructure applications.

{{< /blocks/section >}}
{{< blocks/value title="Visibility" image="placeholder" image_position="left" >}}
  • Deployment pipeline UI shows clarify what is happening
  • Separate logs viewer for each individual deployment
  • Realtime visualization of application state
  • Deployment notifications to slack or webhook endpoints
  • Insight shows the delivery performance
{{< /blocks/value >}} {{< blocks/value title="Automation" image="placeholder" image_position="right" >}}
  • Automated deployment analysis based on metrics, logs, emitted requests
  • Automatically roll back to the previous state as soon as analysis or a pipeline stage fails
  • Automatically detect configuration drift to notify and render the changes
{{< /blocks/value >}} {{< blocks/value title="Secure" image="placeholder" image_position="left" >}}
  • Support single sign-on and role-based access control
  • Credentials are not exposed outside the cluster and not saved in the control-plane
  • Piped makes only outbound requests and can run inside a restricted network
{{< /blocks/value >}} {{< blocks/value title="Multi-provider & Multi-Tenancy" image="placeholder" image_position="right" >}}
  • Multiple application kinds on multi-cloud including Kubernetes, Terraform, CloudRun, Lambda
  • Multiple analysis providers including Prometheus, Datadog, Stackdriver, and more
  • Easy to operate multi-cluster, multi-tenancy by separating control-plane and piped
